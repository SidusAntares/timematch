# v2.3 结果分析

## 1. 版本定位
`v2.3` 是在 `v2.2` 的双路径框架之上，第一次正式引入“结构成分分解”的版本。

与 `v2.3pre` 的差异在于：
- `v2.3pre`：更像黑盒 expert routing
- `v2.3`：显式拆成 `Δh_shape / Δh_phase / Δh_disc`
- 域判定指标不再只是路由黑盒 expert，而是直接决定结构成分权重

## 5. 结果解读
### 5.1 v2.3 初版
`v2.3` 没有超过 `v2.2.2`，也没有超过 baseline。

但它不是完全无效：
- 在一些任务上仍然优于 baseline，例如：
  - `30TXT -> 32VNH`
  - `31TCJ -> 30TXT`
  - `32VNH -> 30TXT`
  - `32VNH -> 31TCJ`
  - `33UVP -> 32VNH`

说明“结构成分分解”的方向本身并没有错。

### 5.2 v2.3.1 轻改
`v2.3.1` 的平滑映射 + phase 保守化，并没有把 `v2.3` 拉上去，反而进一步下降。

这说明：
- 问题不只是“α 太硬”
- 也不只是“phase 成分太强”
- 更可能的核心问题是：
  - **当前 domain signature 到 component 的映射关系本身还不够准确**

## 6. 当前判断
`v2.3` 的意义在于：
- 建立了一个更可解释的 source reshaper 形式
- 把域判定指标、结构动作、残差成分之间的关系理顺了

但就当前性能看：
- `v2.3` 和 `v2.3.1` 都还不足以替代 `v2.2.2`
- `v2.2.2` 仍然是当前正式主线最优版本

## 7. 下一步建议
如果继续推进 `v2.3`，不建议继续只做这类 α 平滑调参。
更值得做的是：
- component ablation
  - 只开 `shape`
  - 只开 `phase`
  - 只开 `disc`
  - 或做小规模组合
- 重新分析：
  - 哪个 component 真正有效
  - 哪个 component 在拖后腿
- 之后再重新设计“指标 -> component weight”的映射关系

结论：
> `v2.3` 方向成立，但当前实现还未达到超过 `v2.2.2` 的程度。  
> 当前正式主线仍应保持为 `v2.2.2`。
