# `sourcephasecompact + timematchtgtphasecompact` 第七版结果分析

## 1. 实验说明

第七版对应：

- source 阶段：`sourcephasecompact`
- DA 阶段：`timematchtgtphasecompact`
- 任务设置：`closed-set`
- augmentation：`no shift aug`

这是当前版本线中第一次明确采用：

- **source-side structure regularization**
- **target pseudo-label structure regularization**

的联合方案。

---

## 2. 整体平均结果

第七版 12 个任务平均 `macro-F1` 为：

- baseline：`0.655286`
- `V2`：`0.658547`
- `V7`：`0.656975`

因此：

- `V7 - baseline = +0.001689`
- `V7 - V2 = -0.001572`
- `V7 - V6 = +0.025758`
- `V7 - V5 = +0.033814`

这说明：

> **第七版已经进入强版本区间。**

它虽然还没有超过第二版 `V2`，但已经：

- 明显优于第五版和第六版
- 高于 baseline
- 也高于第三、四版

---

## 3. 在整个版本序列中的位置

按平均 `macro-F1` 排名：

1. `V2 = 0.658547`
2. `V7 = 0.656975`
3. baseline `= 0.655286`
4. `V1 = 0.655208`
5. `V3 = 0.649766`
6. `V4 = 0.647990`
7. `V6 = 0.631217`
8. `V5 = 0.623161`

也就是说：

> **第七版是当前第二好的版本。**

这一定义很重要，因为它说明：

- 第七版不是一个边缘改动
- 也不是仅在局部任务上偶然起效
- 它已经把“源域约束 + 目标域伪标签结构”这条线推到了接近最优版本的位置

---

## 4. 与第六版的比较

第六版和第七版的差别不只是数值强弱，而是主线逻辑是否完整。

平均结果：

- `V6 = 0.631217`
- `V7 = 0.656975`
- 提升：`+0.025758`

这已经不是轻微波动，而是非常明确的整体跃升。

因此第七版说明：

> **target-side structure line 之所以在第六版仍然偏弱，  
> 很可能不是因为这条线本身必然无效，  
> 而是因为它缺少 source-side structure foundation。**

也就是说，第七版实质上验证了：

- 单独 target 结构线不强
- 但与 source-side compactness 联合后，它就能回到强版本区间

---

## 5. 与第二版的比较

第二版 `V2` 仍然是当前均值最好的版本：

- `V2 = 0.658547`
- `V7 = 0.656975`
- 差值：`-0.001572`

这意味着：

> **第七版还没有正式超越第二版。**

但这个差距已经很小，而且不是单边落后。

从 12 个任务逐项看：

- 第七版优于第二版：`6` 个任务
- 第七版低于第二版：`6` 个任务

因此更准确的判断不是：

- 第七版整体不如第二版

而是：

> **第七版和第二版已经处在同一梯队，只是当前稳定性仍略逊于第二版。**

## 7. 第七版最明显的亮点

第七版最有价值的地方，不在于它把所有任务都推高了，而在于：

> 它在一部分任务上显示出了比第二版更强的上冲能力。

最明显的亮点包括：

- `32VNH -> 33UVP`：`+0.048266`
- `33UVP -> 32VNH`：`+0.040147`
- `33UVP -> 30TXT`：`+0.025415`
- `30TXT -> 33UVP`：`+0.023766`

这说明：

- 第七版并不是简单复制第二版的效果
- 它在某些跨域方向上，确实额外利用到了 target 伪标签结构信息

因此第七版的价值不是：

- “均值略高于 baseline”

而更在于：

> **它首次证明 target 伪标签结构线在合适的 source foundation 上可以转化成真实增益。**

---

## 8. 第七版还存在的问题

第七版当前仍然没有完全压过第二版，主要问题体现在：

- `31TCJ -> 30TXT`
- `31TCJ -> 32VNH`
- `31TCJ -> 33UVP`

也就是说，`31TCJ` 作为 source 时，第七版还不够稳。

这说明当前版本仍然存在：

- source-sensitive
- direction-sensitive

的问题。

更具体地说：

> 第七版已经具备更强的结构表达能力，但这种能力还没有被稳定控制住。  
> 它能带来更高上限，也更容易在某些 source/target 组合上产生回撤。

---

## 9. 第七版真正说明了什么

第七版最重要的结论不是：

- target 伪标签结构约束已经全面成功

而是：

> **target 伪标签结构约束不是天然错误的；  
> 它之所以在第五版里表现差，核心问题之一是缺少 source-side structure foundation。**

这一点非常关键，因为它直接改变了对第五、第六版的解释。

从第七版回头看：

- `V5` 失败，并不代表 target 结构线本身完全不可用
- `V6` 偏弱，也不代表 target 结构线没有潜力
- 更准确地说：
  - target 结构线需要和 source 结构线配套设计

---

## 10. 当前结论

第七版的最终评价最适合概括为：

> **第七版已经证明“源域约束 + 目标域伪标签结构”是一条成立的联合主线，  
> 它明显优于第五版和第六版，并进入当前最强版本梯队；  
> 但从整体均值看，它仍然略低于第二版，因此还不能说已经完成对第二版的稳定超越。**

因此当前最合理的版本判断是：

- 如果追求**最稳的默认版本**：第二版仍然最好
- 如果追求**最值得继续推进的联合结构版本**：第七版最值得保留

换句话说：

> 第七版不是终点，但它已经把“target structure line 是否值得保留”这个问题，  
> 从原来的不确定，推进到了明确的“值得继续做”。
