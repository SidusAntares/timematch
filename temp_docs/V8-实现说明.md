# `sourcephasecompact + timematchtgtphaseconsistency` 第八版实现说明

## 1. 版本定位

第八版是在第七版基础上做的一次关键消融。

如果用一句话概括，第八版就是：

> **保留第七版的 source-side structure shaping，只把 target-side 约束切回第六版的无监督 consistency。**

因此它的核心标签是：

- **源域约束**
- **非伪标签目标结构**

---

## 2. 与第六版和第七版的关系

第八版的设计目的，不是简单回退到第六版，也不是重复第七版，而是补齐一个关键中间点。

### 第六版

- 无 source 结构约束
- target 侧使用无监督 `phase consistency`

### 第七版

- 有 source 结构约束
- target 侧使用伪标签 `phase compactness`

### 第八版

- 有 source 结构约束
- target 侧使用无监督 `phase consistency`

因此第八版真正要回答的问题是：

> **当 source-side structure foundation 已经建立后，第六版那种无监督 target consistency 还能不能真正工作起来？**

---

## 3. 方法流程

第八版依然分成两个阶段。

### 阶段一：source-only structure shaping

先运行：

- `sourcephasecompact`

也就是：

- 在 source-only supervised training 阶段加入 source `phase compactness`
- 使用 `uniform 5-phase`
- 得到一个结构更规整的 source model

### 阶段二：DA-stage target unsupervised structure shaping

再以这个 source model 为初始化，进入 DA 阶段，运行：

- `timematchtgtphaseconsistency`

也就是：

- 保留普通 `TimeMatch` 的 source supervised loss
- 保留普通 `TimeMatch` 的 target pseudo-label consistency loss
- 额外加入第六版的 target 无监督 `phase consistency`

整体流程可以写成：

```text
sourcephasecompact  ->  timematchtgtphaseconsistency
```

---

## 4. 第八版和第六版的本质区别

第六版和第八版使用的是同一类 target 结构约束，但初始基础不同。

### 第六版

- source 初始化：普通 baseline source-only
- DA 阶段：直接使用 target 无监督 `phase consistency`

### 第八版

- source 初始化：先经过 `sourcephasecompact`
- DA 阶段：再使用 target 无监督 `phase consistency`

因此第八版新增的重点不是：

- “换一个新的 target loss”

而是：

> **在一个已经做过 source-side structure shaping 的表示基础上，再重新测试第六版 target consistency。**

---

## 5. 设计动机

第八版的动机来自前面几版结果的组合判断：

1. `V2` 说明：
   - source-only compactness 是当前最稳的 source-side 正结果版本
2. `V6` 说明：
   - 无监督 target consistency 比第五版伪标签 compactness 更合理
   - 但单独使用时仍然不够强
3. `V7` 说明：
   - 当 source-side structure foundation 存在时，target-side structure line 可以重新变强

因此一个自然的问题就是：

> 第七版之所以强，到底是因为“target 端用了伪标签 compactness”，  
> 还是因为“source-side compactness 先把共享表示空间整理好了”？

第八版就是对这个问题的直接实验回答。

---

## 6. 第八版要回答的核心问题

第八版真正要回答的是：

> **target-side structure line 想成立，关键前提到底是不是 source-side foundation？**

如果第八版明显优于第六版，就说明：

- 第六版的问题并不在于无监督 consistency 本身
- 更大的问题在于它缺少 source-side structure foundation

如果第八版还能接近第七版，就进一步说明：

- target 端不一定非得依赖伪标签
- 无监督结构约束在合适的 source 初始化上也能成立

---

## 7. 第八版在版本序列中的意义

如果说：

- `V5` 是 target 伪标签结构线的首次激进尝试
- `V6` 是 target 无监督结构线的修正版
- `V7` 是 source 约束与 target 伪标签结构的首次合流

那么：

> `V8` 就是 source 约束与 target 无监督结构的首次合流。

这使得第八版在整个版本演化中非常关键，因为它补齐了一个此前缺失的消融点：

- `V6`：无 source 约束 + 无监督 target 约束
- `V7`：有 source 约束 + 伪标签 target 约束
- `V8`：有 source 约束 + 无监督 target 约束

因此第八版不只是“又做一个版本”，而是在方法论上补全了 source line 和 target line 的实验结构。
