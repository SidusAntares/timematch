# v2.2 结果分析

## 1. 版本定位
`v2.2` 是 `v2` 系列中第一个真正把新路线拉回 baseline 档位的版本。

相对 `v2.1`，`v2.2` 的关键变化是：
- 保留 source reshaper
- 采用 `raw source path + reshaped source path` 双路径训练
- 用 relation consistency 约束 raw / reshaped 两条 source 路径
- target 仍然只走 raw path，不让下游只认 reshaped feature

这使得 `v2.2` 不再只是“可行性验证”，而是开始具备竞争力。

## 2. 实验配置
完整 12 任务实验中，对比了两组主要配置：

### 配置 A
- `residual_temporal_conv_s0.10_k3_r0.05_dual_c1.00_rel0.05`

### 配置 B
- `residual_temporal_conv_s0.15_k3_r0.05_dual_c1.00_rel0.10`

## 3. 整体结果
12 个任务平均 `macro_f1`：

- baseline: **0.6553**
- `v2.1` 最好一组 `s0.10`: **0.6339**
- `v2.2-A`: **0.6550**
- `v2.2-B`: **0.6464**

结论：
- `v2.2-A` 基本追平 baseline
- `v2.2-A` 明显优于 `v2.1`
- `v2.2-B` 不如 `v2.2-A`

当前 `v2.2` 主候选配置：
- **`residual_temporal_conv_s0.10_k3_r0.05_dual_c1.00_rel0.05`**

## 5. 相比 v2.1 的改进
`v2.2` 的双路径设计成功修复了 `v2.1` 中“source-only reshaper 可能让下游更偏向 reshaped feature”的问题。

典型提升包括：
- `30TXT -> 31TCJ`: `0.7310 -> 0.7896`
- `31TCJ -> 30TXT`: `0.7550 -> 0.7745`
- `31TCJ -> 32VNH`: `0.4738 -> 0.5298`
- `32VNH -> 30TXT`: `0.5347 -> 0.5831`
- `32VNH -> 33UVP`: `0.5808 -> 0.6480`
- `33UVP -> 31TCJ`: `0.5616 -> 0.6088`
- `33UVP -> 32VNH`: `0.6829 -> 0.7663`

## 6. 调参结果补充
`v2.2` 后续又做了 `v2.2.x` 小范围调参，最佳配置更新为：
- **`v222_s010_rel003`**
- 即：`strength = 0.10`, `dual_relation_trade_off = 0.03`

该配置的 12 任务平均 `macro_f1` 为：
- **0.6787**

因此，当前 `v2.2` 系列真正的最优主线配置是：
- **`v2.2.2 = v222_s010_rel003`**

## 7. 结论
`v2.2` 证明：
- 双路径 source 结构诱导是正确方向
- relation consistency 需要较保守的强度
- `raw anchor + reshaped auxiliary` 是 `v2` 系列站住的关键

当前在 `v2` 系列中，`v2.2.2` 仍然是正式主线最优版本。
