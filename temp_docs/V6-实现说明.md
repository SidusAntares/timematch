# `timematchtgtphaseconsistency` 第六版实现说明

## 1. 版本定位

第六版是在第五版之后，对 target-side structure line 做的一次明确修正。

第五版的问题已经比较清楚：

- 它直接依赖 teacher 伪标签
- 对高置信 target 样本施加 class-conditional `phase compactness`
- 一旦伪标签分区本身有偏差，就会把错误结构进一步收紧

因此，第六版不再沿用“显式伪标签类条件 compactness”这条路线，而改成：

> **不使用伪标签分组信息，而是用非伪标签方式约束 target 域自身结构一致性。**

所以，第六版的核心标签可以概括为：

- **无源域约束**
- **非伪标签目标结构**

---

## 2. 与前几版的关系

前五版主线可以分成两段：

1. `V1 ~ V4`
   - 主要围绕 source-side structure regularization
   - 关注点是：source 如何变成更好的 source

2. `V5`
   - 首次正面转向 target-side structure
   - 但直接使用伪标签类结构，结果整体明显负增益

第六版的出发点正是：

> 如果 target-side structure 这条线值得保留，  
> 那么要先去掉最危险的环节，也就是“显式伪标签类条件收紧”。 

因此第六版不是回到 source 线，而是：

> **在 target 结构主线内部，从“伪标签类条件 compactness”切到“非伪标签结构 consistency”。**

---

## 3. 方法动机

第六版背后的判断是：

- 第五版出问题，不一定说明 target 结构完全不能做
- 更可能说明：
  - target 结构约束不能过度依赖当前 teacher 的类划分
  - 不能把尚不可靠的伪标签直接当成结构监督真值

因此一个更温和的版本应该满足：

1. 仍然在 DA 阶段显式约束 target 结构
2. 但不要求 target 样本先被硬分进某一类
3. 让 target 结构约束尽量少受伪标签噪声直接驱动

第六版就是沿着这个逻辑设计的。

---

## 4. 方法位置

第六版仍然作用在 **TimeMatch 域适应阶段**，而不是 source-only 训练阶段。

整体流程为：

1. 使用普通 baseline source-only 模型初始化 `TimeMatch`
2. 保留原始 TimeMatch 的：
   - source supervised loss
   - target pseudo-label consistency loss
3. 再额外加入：
   - target-domain phase consistency / structure consistency regularizer

因此，第六版总体上仍然属于：

> **“target-side structure during adaptation”**

而不是回到 source-side 结构塑形。

---

## 5. 与第五版的本质区别

第五版和第六版的差别，不只是名字变化，而是结构假设发生了变化。

### 第五版

- 使用 target 高置信伪标签
- 做 class-conditional phase compactness
- 本质上是在说：
  - “teacher 现在给出的同类样本应该更紧”

### 第六版

- 不再直接依赖伪标签类别分组
- 改为从更弱、更连续的方式约束 target 结构
- 本质上是在说：
  - “target 表示在 phase 结构上应该更一致、更稳定”

也就是说：

> 第五版约束的是 target 的**伪标签类别结构**，  
> 第六版约束的是 target 的**非伪标签结构一致性**。

---

## 6. 为什么第六版不加入 source 约束

第六版是刻意不加 source-side structure regularization 的。

这样做的目的是保证实验解释足够清楚：

- 如果第六版优于第五版
- 那么可以更直接地把差异归因到：
  - target 结构约束方式变了

而不是因为：

- source 训练也一起变了

所以第六版的定位是一个比较干净的对照版本：

> **只改 target 结构形式，不改 source 结构。**

---

## 7. 第六版要回答的问题

第六版想回答的不是：

- source 结构该怎么做

而是：

> **如果保留“target-side structure”这条主线，但去掉伪标签类条件收紧，结果会不会更合理？**

它的实验价值主要体现在两个方面：

1. 判断第五版的失败是不是主要来自“伪标签直接参与结构收紧”
2. 判断 target 结构线本身是否还有继续保留的必要

---

## 8. 第六版在整个版本序列中的意义

从方法演化位置上看，第六版是一个非常关键的过渡版本：

- 它不是最终强版本
- 但它能帮助我们判断：
  - 问题到底出在 target-side structure 这条线本身
  - 还是出在第五版那种过强、过硬的伪标签结构监督形式

因此，第六版的意义更接近：

> **target structure line 的去噪修正版 / 诊断版。**

如果它比第五版更好，就说明：

- target 结构线还有保留价值
- 只是监督形式需要更谨慎

如果它仍然明显弱于 baseline，就说明：

- target 结构线即使去掉显式伪标签收紧，也还没有形成稳定正增益

这正是第六版的实验定位。
