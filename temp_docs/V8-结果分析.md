# `sourcephasecompact + timematchtgtphaseconsistency` 第八版结果分析

## 1. 实验说明

第八版对应：

- source 阶段：`sourcephasecompact`
- DA 阶段：`timematchtgtphaseconsistency`
- 任务设置：`closed-set`
- augmentation：`no shift aug`

它是当前版本线中第一次明确采用：

- **source-side structure regularization**
- **target-side unsupervised structure regularization**

的联合方案。

---

## 2. 整体平均结果

第八版 12 个任务平均 `macro-F1` 为：

- baseline：`0.655286`
- `V6`：`0.631217`
- `V7`：`0.656975`
- `V8`：`0.655625`

因此：

- `V8 - baseline = +0.000339`
- `V8 - V6 = +0.024408`
- `V8 - V7 = -0.001350`
- `V8 - V2 = -0.002922`

这说明：

> **第八版已经完全脱离第六版的弱版本区间，进入了 baseline / V7 所在的强版本区间。**

---

## 3. 在整个版本序列中的位置

按平均 `macro-F1` 排名：

1. `V2 = 0.658547`
2. `V7 = 0.656975`
3. `V8 = 0.655625`
4. baseline `= 0.655286`
5. `V1 = 0.655208`
6. `V3 = 0.649766`

8. `V6 = 0.631217`
9. `V5 = 0.623161`

也就是说：

> **第八版是当前第三好的版本。**

这一定义很重要，因为它说明第八版不是一个边缘修补版本，而是已经进入了当前主线强版本区间。

---

## 4. 与第六版的比较

第六版和第八版的主要区别只有一件事：

- 第六版没有 source-side structure foundation
- 第八版有 `sourcephasecompact`

平均结果：

- `V6 = 0.631217`
- `V8 = 0.655625`
- 提升：`+0.024408`

这是一个很清楚的整体跃升。

从任务层面看，第八版相对第六版提升最明显的包括：

- `31TCJ -> 30TXT`
- `31TCJ -> 32VNH`
- `33UVP -> 32VNH`

因此第八版说明：

> **第六版 target consistency 本身不是错的，它的问题更像是缺少 source-side structure foundation。**

换句话说：

- 无监督 target consistency 单独跑时不够强
- 但一旦前面加上 source compactness，它就能真正工作起来

---

## 5. 与第七版的比较

第七版和第八版的差别在 target 端：

- `V7`：target 伪标签 `phase compactness`
- `V8`：target 无监督 `phase consistency`

平均结果：

- `V7 = 0.656975`
- `V8 = 0.655625`
- 差值：`-0.001350`

这个差距非常小，基本可以看作同一强度档次。

这说明：

> **在 source-side compactness 已经成立后，target 端不一定非得依赖伪标签，使用无监督 consistency 也能达到几乎同样的效果。**

因此，第八版给出的最重要新信息不是“它超过了第七版”，而是：

> **第七版的成功，并不完全依赖 target 伪标签结构约束。**

---

## 6. 第八版真正证明了什么

第八版最重要的实验意义，是把下面这件事说清楚了：

- `V6` 失败，不代表无监督 target structure line 无效
- `V5` 失败，也不代表 target structure line 本身不能做

更准确地说：

> **target-side structure line 是否成立，很大程度取决于 source-side structure 是否已经先被整理好。**

第八版正是这个判断的直接证据。

它说明：

1. source-side compactness 不是只对 source-only 有价值
2. 它还会改变 target-side regularizer 的可用性
3. 当 source foundation 存在时，第六版那条无监督 consistency 线可以重新进入强版本区间

---

## 7. 第八版与第七版的结构意义差别

虽然第七版和第八版数值非常接近，但它们的方法学含义不同。

### 第七版说明

- source compactness 打底后
- target 伪标签结构约束可以成立

### 第八版说明

- source compactness 打底后
- target 无监督结构约束也可以成立

因此：

> **第七版证明了“伪标签 target structure”可以成立，  
> 第八版进一步证明了“无监督 target structure”也可以成立。**

这意味着 target 结构这条线，不再只有一条单一路径，而是至少已经分化出两条可行路线：

- `V7`：伪标签结构路线
- `V8`：无监督结构路线

---

## 8. 当前结论

第八版最合适的总结是：

> **第八版是一个非常成功的消融版本。**
> 它清楚地证明了：source-side structure shaping 才是 target structure line 能否成立的重要前提之一。  
> 当这一前提满足时，即使不用伪标签，target 无监督 consistency 也能够进入强版本区间，并取得与第七版非常接近的结果。

因此当前最合理的判断是：

- 如果追求**最稳的默认版本**：`V2` 仍然最好
- 如果追求**最值得继续推进的联合结构版本**：`V7` 和 `V8` 都值得保留
- 如果追求**更干净的方法解释**：`V8` 的实验价值尤其高，因为它证明了 target 端并不一定要依赖伪标签
