# `sourcephasecompact + timematchtgtphasecompact` 第七版实现说明

## 1. 版本定位

第七版是当前这条实验主线里，第一次明确把：

- **source-side structure shaping**
- **target-side pseudo-label structure shaping**

放进同一个版本里联合设计。

如果用一句话概括，第七版就是：

> **先把 source 变得更有结构，再去约束 target 的伪标签结构。**

因此它的核心标签是：

- **源域约束**
- **伪标签目标结构**

---

## 2. 与第六版的关系

第六版已经说明了一件事：

- target 结构线如果完全不依赖 source-side structure foundation
- 即使去掉“显式伪标签类条件 compactness”这种过强监督
- 整体也仍然不够强

因此第七版的出发点是：

> target-side structure 之所以在第五、第六版里表现不稳，  
> 也许不是因为 target structure 这条线本身完全无效，  
> 而是因为它缺少一个已经被显式整理过的 source 表示基础。

所以第七版不是简单回到第五版，而是：

1. 先引入第二版中已经验证过有效的 source-side structure step
2. 再在此基础上重新启用 target 伪标签结构约束

这使得第七版成为一个真正的联合版，而不是单侧修补版。

---

## 3. 方法流程

第七版的实验流程分成两个阶段。

### 阶段一：source-only structure shaping

先跑：

- `sourcephasecompact`

也就是：

- 在 source-only supervised training 阶段加入 source `phase compactness`
- 使用 `uniform 5-phase`
- 得到一个结构更规整的 source model

### 阶段二：DA-stage target pseudo-label structure shaping

再以这个 source model 为初始化，进入 DA 阶段，运行：

- `timematchtgtphasecompact`

也就是：

- 保留普通 `TimeMatch` 的 source supervised loss
- 保留普通 `TimeMatch` 的 target pseudo-label consistency loss
- 再额外加入 target 高置信伪标签样本的 `phase compactness`

因此整体流程可以写成：

```text
sourcephasecompact  ->  timematchtgtphasecompact
```

---

## 4. 第七版和第五版的本质区别

第五版和第七版都用了 target 高置信伪标签结构约束，但它们不是同一类版本。

### 第五版

- source 初始化：普通 baseline source-only
- DA 阶段：直接加入 target pseudo-label phase compactness

### 第七版

- source 初始化：先经过 `sourcephasecompact`
- DA 阶段：再加入 target pseudo-label phase compactness

因此，第七版真正新增的不是：

- “又一次做 target compactness”

而是：

> **在一个已经做过 source-side structure shaping 的初始化上，再去做 target pseudo-label structure shaping。**

这意味着第七版在结构逻辑上不再是：

- “直接对 target 当前 teacher partition 施压”

而更像是：

- “先让 source 表示基础变规整，再让 target 伪标签结构在这个基础上收紧”

---

## 5. 方法动机

第七版背后的判断来自前几版结果的综合：

1. `V2` 说明：
   - source-only compactness 是当前最稳的 source-side 正结果版本

2. `V5` 说明：
   - 直接 target pseudo-label compactness 会整体负增益

3. `V6` 说明：
   - target structure line 不是完全不能做
   - 但单独做仍然偏弱

所以一个很自然的综合假设是：

> target pseudo-label structure 之所以在第五版容易出问题，  
> 可能不是因为“伪标签结构”本身毫无价值，  
> 而是因为它被施加在一个尚未被 source-side 显式整理过的共享表示空间上。

第七版就是对这个假设的第一次系统验证。

---

## 6. 第七版要回答的问题

第七版真正要回答的是：

> **当 source-side structure foundation 先被建立后，  
> target pseudo-label class-conditional phase compactness 还能不能从负结果变成可用结果？**

这比第五版和第六版的问题都更进一步，因为它不再把 source 和 target 结构线割裂开看，而是开始把它们当成一个联合系统来设计。

---

## 7. 第七版在整个版本序列中的意义

如果说：

- `V5` 是 target structure line 的第一次激进尝试
- `V6` 是 target structure line 的去噪修正版

那么：

> `V7` 就是 source line 和 target line 的第一次真正合流。

它的实验意义非常重要，因为它不只是再试一个新 loss，而是在测试一个新的主线判断：

> **target-side structure regularization 也许只有在 source-side structure 已经先被整理好的前提下，才可能真正有用。**

因此，第七版在版本演化中的角色是：

- 不是简单的“第五版修补”
- 而是一个新的联合结构版本

这也是为什么第七版比第五、第六版更值得重点分析。
