# `timematchtgtphaseconsistency` 第六版结果分析

## 1. 实验说明

第六版对应：

- 方法名：`timematchtgtphaseconsistency`
- 任务设置：`closed-set`
- augmentation：`no shift aug`
- 核心特点：
  - **无 source-side structure regularization**
  - **使用非伪标签方式约束 target 域结构**

这一版最直接的比较对象是第五版 `timematchtgtphasecompact_p5`，因为：

- 两者都属于 target-side structure line
- 主要差别在于：
  - 第五版：显式伪标签 class-conditional compactness
  - 第六版：非伪标签 target structure consistency

---

## 2. 与 baseline 的整体关系

第六版 12 个任务的平均 `macro-F1` 为：

- baseline：`0.655286`
- `V6`：`0.631217`
- 相对 baseline：`-0.024069`

这说明：

> **第六版虽然比第五版更合理，但整体仍然明显低于 baseline。**

因此第六版不能被看作已经成功的正结果版本。

---

## 3. 与第五版的整体比较

平均结果如下：

- `V5`：`0.623161`
- `V6`：`0.631217`
- `V6 - V5 = +0.008056`

这说明：

> **第六版确实修复了第五版的一部分问题。**

也就是说：

- 第五版最危险的部分，确实很可能就是“用伪标签类划分直接做 compactness”
- 改成非伪标签结构约束之后，整体损伤有所减轻

但与此同时：

> 这个修复幅度还不够大，尚不足以把第六版拉回 baseline 水平。

## 5. 第六版真正改好了什么

从上表看，第六版的改进并不是平均摊开的，而是集中出现在几个任务上：

### 明显提升的任务

- `33UVP -> 30TXT`：`+0.081933`
- `32VNH -> 33UVP`：`+0.034628`
- `33UVP -> 32VNH`：`+0.017795`

这说明第六版在部分迁移方向上，确实比第五版稳定得多。

也就是说：

> 当不再把 teacher 当前类划分直接当成结构真值时，  
> target-side structure loss 至少不会像第五版那样那么容易“把错误越压越紧”。

---

## 6. 但第六版仍然没有解决什么

尽管比第五版更好，第六版仍然存在两个明显问题：

### 6.1 它仍然没有形成稳定总体增益

相对 baseline 仍然是：

- `-0.024069`

所以它还不是：

- 一个可直接保留为主线最终版的方法

### 6.2 它仍然缺少 source-side 结构先验

第五版和第六版有一个共同点：

- 都没有先对 source 侧做结构整理

因此 target 结构约束实际上是在一个尚未被显式结构化的 shared representation 上工作。

这很可能导致：

- target 结构线即使形式变温和了
- 也缺少足够好的表示基础

这也是为什么第六版能比第五版好，但依旧不够强。

---

## 7. 第六版在版本演化中的结论

第六版最重要的结论不是：

- target structure line 已经成功

而是：

> **第五版的失败并不意味着 target structure 完全不能做；  
> 更准确地说，第五版失败的核心原因之一，确实是“直接使用伪标签类条件 compactness”过于激进。**

换句话说：

- `V5 -> V6` 的改善
  - 证明了 target structure 监督形式很关键
- 但 `V6` 仍低于 baseline
  - 说明 target structure 这条线仅靠自己还撑不起来

因此第六版的合理定位是：

> **一个必要的修正版和过渡版。**

它把我们从：

- “目标结构线当前明显有问题”

推进到了：

- “目标结构线可以保留，但必须与更好的 source-side structure foundation 联合设计”

这也正好为第七版提供了方法学动机。
