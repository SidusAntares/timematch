# v2.5.2e：source-side defect gate 实验记录

## 版本目的

`v2.5.2e` 试图解决当前闭环中缺失的“源域结构约束自适应”问题。核心假设是：不同源域距离适合域适应的结构形态不同，因此不应该对 trend、segment inter、boundary window、reshaper、dual relation 使用完全固定的约束力度。

这一版没有使用历史最优结果去定义 ideal profile，也没有使用早期 loss 大小做预算分配，而是尝试 **source-side defect gate**：先在源域训练前 10 epoch 统计特征空间中的缺陷信号，再给各结构成分一个保守倍率。

实现要点：

- 前 10 epoch 维持 `v2.4.3b` 的基础结构权重，累积 profile。
- 使用 `intra_loss` 作为基础锚点，计算 trend / segment inter 等相对缺陷。
- 使用 raw feature temporal roughness 作为 reshaper 缺陷信号。
- 使用 raw/reshaped prediction disagreement 与 dual relation loss 作为 dual relation 缺陷信号。
- 第 10 epoch 后一次性设置 component factors。
- 默认范围为 `[0.8, 1.2]`，默认关闭后期 controller。

## 结论

`v2.5.2e` 判定为失败。

主要原因不是单纯平均分低，而是失败模式很清楚：

- 相比 `v2.4.3b 100ep`，平均下降约 `2.0` 个 macro F1。
- 相比 `v2.4.3b 50ep`，平均也略低，说明即使按实际 50 epoch 对齐也没有收益。
- 最大负向任务是 `31TCJ->32VNH`，从 `68.6` 降到 `49.1`，说明 gate 会严重破坏某些源-目标组合需要的结构。
- gate 输出几乎在所有源域饱和到相同方向，未实现真正的 source-adaptive。
- 这种 defect 指标仍然只看源域自身，不知道某个结构缺陷对目标域适应是有利还是有害。

## 反思

当前 defect gate 的问题是：它把“缺陷大”直接解释为“应该加大对应约束”。但域适应里不一定成立。

例如：

- segment/boundary 的相对缺陷大，可能表示源域结构确实需要加强，也可能表示该结构本来就是源域特有模式，强行压缩会降低迁移。
- dual_relation/disagreement 大，可能表示 raw/reshaped 分支需要一致，也可能表示 reshaper 正在制造对目标域有用的结构扰动。
- roughness 大，可能表示噪声，也可能表示作物物候关键转折更明显。

所以 source-side 自适应不能只依赖“源域缺陷强度”。它至少需要回答一个更具体的问题：这个成分的调整是否提升了当前源域形成可迁移表示的倾向。

## 下一步

当前应回退到 `v2.4.3b` 干净代码，把 `v2.5.2e` 作为失败版本归档。

更合理的闭环方向是：

- 源域自适应：不要继续做规则 gate；如果要做神经网络式调控，需要让调控器看到训练过程中结构成分对分类/表示稳定性的边际影响，而不是只看缺陷绝对值。
- 目标域自适应：继续保留 checkpoint selection 作为主线，因为它直接对“同一源域不同目标域需要不同训练状态”建模。
- DA 阶段任务自适应：可以考虑利用 TimeMatch 阶段的伪标签置信度、shift 稳定性、teacher-student 一致性、目标域高置信样本结构等信号，微调 reshaper 或结构成分，而不是让源域 gate 一次性决定全部任务。
